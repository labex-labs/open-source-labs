Git 은 개발자가 코드베이스에 적용된 변경 사항을 추적할 수 있도록 하는 강력한 버전 관리 시스템입니다. Git 의 주요 기능 중 하나는 기록을 다시 작성하는 기능으로, 이전 커밋에 변경 사항을 적용해야 할 때 유용할 수 있습니다. 그러나 기록을 다시 작성하는 것은 원격 저장소에 변경 사항을 푸시하려고 할 때 문제를 일으킬 수 있습니다. 이 랩에서는 로컬에서 기록을 다시 작성한 후 원격 브랜치를 업데이트하는 방법을 배우게 됩니다.
