# ファストフォワードマージの無効化

デフォルトでは、Git は分岐したコミットがないブランチをマージするためにファストフォワードマージを使用します。これは、新しいコミットがないブランチがある場合、Git が単にマージ先のブランチのポインタをマージ元のブランチの最新コミットに移動することを意味します。これは一部のケースで便利ですが、特に複数の貢献者がいる大規模なプロジェクトで作業するときには問題を引き起こす場合もあります。たとえば、2 人の開発者が同じブランチで作業して両方とも変更を加えた場合、ファストフォワードマージは解決しにくい競合を引き起こす可能性があります。

## タスク

ファストフォワードマージを無効にするには、`https://github.com/labex-labs/git-playground` のリポジトリを使用しましょう。

1. ディレクトリに移動して ID を設定します。
2. `my-branch` という名前のブランチを作成して切り替え、`hello.txt` ファイルを作成してそこに "hello,world" を追加し、ステージングエリアに追加して "Added hello.txt" というメッセージでコミットします。
3. すべてのブランチに対してファストフォワードマージを無効にします。
4. `master` ブランチに戻り、`my-branch` ブランチをマージし、テキストを変更せずに保存して終了します。

これで、ファストフォワードが可能であっても Git は常にマージコミットを作成します。

```shell
commit 6e17a776ab51a89ace069614b0caf1c07915a92c (HEAD -> master)
Merge: ec5ea6d 6d7de91
Author: xiaoshengyunan <@users.noreply.github.com>
Date:   Mon Jul 17 13:30:44 2023 +0800

    Merge branch'my-branch'
```
