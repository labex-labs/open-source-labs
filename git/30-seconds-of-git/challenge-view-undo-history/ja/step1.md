# 「元に戻す」履歴を表示する

開発者として、コードベースに対して行った変更を元に戻す必要がある場合があります。Git には、変更を元に戻すためのいくつかの方法が用意されており、たとえば `git reset` や `git revert` コマンドを使用することができます。ただし、特に `git rebase` のような高度なコマンドを使用した場合、行ったすべての操作を追跡するのは難しい場合があります。

## タスク

リポジトリにいくつかの変更を加えたとしましょう。そして、それらの変更を元に戻したいと思います。

1. リポジトリに移動します。
2. 今、ミスを犯したことに気づき、最後のコミットを元に戻したいと思います。
3. もう1つのミスを犯したことに気づき、リセットを元に戻したいと思います。参照ログを表示して、前のコミットのコミットハッシュを見つけます。
4. 前のコミットハッシュが `d22f46b` であることがわかり、このハッシュを使用してリポジトリを前のコミットに戻します。
5. 履歴のコミットレコードを表示して結果を確認します。

以下はステップ3の結果です。これにより、リポジトリで行ったすべての操作（リセットも含む）の一覧が表示されます。

```shell
cf80005 (HEAD -> master) HEAD@{0}: reset: moving to HEAD~1
d22f46b (origin/master, origin/feature-branch, origin/HEAD) HEAD@{1}: clone: from https://github.com/labex-labs/git-playground.git
```
