Git 에서, 레포지토리 (repository) 는 Git 에 의해 추적되는 파일과 폴더의 모음입니다. 파일에 변경 사항을 적용하면 Git 은 파일의 새 버전을 생성하여 레포지토리에 저장합니다. 시간이 지남에 따라 레포지토리는 파일의 이전 버전 및 기타 불필요한 데이터로 인해 복잡해질 수 있습니다. 이는 Git 의 속도를 늦추고 레포지토리 작업을 더 어렵게 만들 수 있습니다. 이 랩에서는 로컬 레포지토리의 성능을 향상시키기 위해 로컬 레포지토리를 최적화하는 방법을 배우게 됩니다.
