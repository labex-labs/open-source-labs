Git es un sistema de control de versiones poderoso que permite a los desarrolladores rastrear los cambios realizados en su repositorio de código. Una de las características clave de Git es la capacidad de reescribir el historial, lo que puede ser útil cuando necesites hacer cambios a commits anteriores. Sin embargo, reescribir el historial puede causar problemas al intentar empujar cambios a un repositorio remoto. En este desafío, aprenderás cómo actualizar una rama remota después de reescribir el historial localmente.
