プロジェクトを行う際、コードベースに対する各変更について、一貫性があり情報が充実したコミットメッセージを付けることが重要です。これにより、他の開発者が変更の目的を理解しやすくなり、必要に応じてバグを特定したり変更を元に戻したりすることが容易になります。この実験では、Git リポジトリのコミットメッセージ テンプレートを設定する方法を学びます。
