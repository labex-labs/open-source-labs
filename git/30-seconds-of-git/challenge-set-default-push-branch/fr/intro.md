Lorsque vous travaillez avec Git, il est courant de pousser vos modifications locales vers un référentiel distant. Par défaut, Git poussera vos modifications vers une branche ayant le même nom que votre branche locale. Cependant, parfois vous souhaiterez peut-être pousser vos modifications vers une autre branche. Dans ce défi, vous allez apprendre à définir le nom de la branche de poussée par défaut sur le nom de la branche locale actuelle.
