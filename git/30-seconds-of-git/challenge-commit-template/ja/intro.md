プロジェクトを行う際、コードベースに対する各変更について、一貫性のある情報豊かなコミットメッセージを作成することが重要です。これにより、他の開発者が変更の目的を理解しやすくなり、バグの特定や必要に応じた変更の元に戻す作業が容易になります。このチャレンジでは、Git リポジトリのコミットメッセージ テンプレートを設定する方法を学びます。
