No Git, um _commit_ é um _snapshot_ das alterações feitas em um repositório. Cada _commit_ tem uma mensagem que descreve as alterações feitas. Às vezes, pode ser necessário criar um _commit_ vazio, sem alterações, para servir como um _placeholder_ ou para acionar um processo de _build_. Neste laboratório, você aprenderá como criar um _commit_ vazio no Git.
