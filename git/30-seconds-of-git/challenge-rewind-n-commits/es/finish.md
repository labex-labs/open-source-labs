# Resumen

Retroceder en los commits es una característica útil de Git que permite a los desarrolladores deshacer cambios y revertir a una versión anterior de su código. Al usar el comando `git reset`, los desarrolladores pueden retroceder fácilmente en sus commits y volver a una versión anterior de su código. En este desafío, has aprendido cómo usar Git para retroceder en los commits y volver a una versión anterior de tu código.
