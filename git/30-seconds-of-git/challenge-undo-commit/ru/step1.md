# Отмена коммита

Предположим, что вы сделали коммит в свой репозиторий Git, но понимаете, что он содержит ошибку. Вы хотите отменить коммит, не переписывая историю репозитория. Как это можно сделать?

## Задачи

Для демонстрации того, как отменить коммит, давайте используем репозиторий из `https://github.com/labex-labs/git-playground`.

1. Перейдите в директорию репозитория и настройте свою GitHub-личность.
2. Просмотрите историю коммитов.
3. Выберите коммит с сообщением "Added file1.txt" и скопируйте его идентификатор.
4. Отмените коммит, и Git откроет текстовый редактор и позволит вам ввести сообщение коммита, оставив стандартное сообщение на месте.
5. Сохраните и закройте текстовый редактор.
6. Посмотрите историю коммитов снова.

Вы должны увидеть новый коммит, который отменяет изменения, внесенные в исходном коммите.

Вот результат выполнения команды `git log`:

```
commit 0d01f357a798f8960959546750d89a7e56a04a44 (HEAD -> master)
Author: xiaoshengyunan <@users.noreply.github.com>
Date:   Mon Jul 24 21:52:43 2023 +0800

    Revert "Added file1.txt"

    This reverts commit cf80005e40a3c661eb212fcea5fad06f8283f08f.
```
