# Ограничение скорости

Задача заключается в том, чтобы ограничить обработку входящих запросов для поддержания качества обслуживания и контроля использования ресурсов.

## Требования

- Язык программирования Go
- Основное понимание горутин, каналов и тикеров

## Пример

```sh
# Запустив нашу программу, мы видим, что первая партия запросов
# обрабатывается раз в ~200 миллисекунд, как и ожидалось.
$ go run rate-limiting.go
request 1 2012-10-19 00:38:18.687438 +0000 UTC
request 2 2012-10-19 00:38:18.887471 +0000 UTC
request 3 2012-10-19 00:38:19.087238 +0000 UTC
request 4 2012-10-19 00:38:19.287338 +0000 UTC
request 5 2012-10-19 00:38:19.487331 +0000 UTC

# Для второй партии запросов мы сразу обрабатываем первые
# 3 из-за ограничения скорости с возможностью всплеска,
# а затем обрабатываем оставшиеся 2 с задержкой ~200 мс каждым.
request 1 2012-10-19 00:38:20.487578 +0000 UTC
request 2 2012-10-19 00:38:20.487645 +0000 UTC
request 3 2012-10-19 00:38:20.487676 +0000 UTC
request 4 2012-10-19 00:38:20.687483 +0000 UTC
request 5 2012-10-19 00:38:20.887542 +0000 UTC
```
