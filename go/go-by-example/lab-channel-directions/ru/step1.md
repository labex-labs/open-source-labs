# Направления каналов

Задача, которую необходимо решить в этом лабораторном практикуме, заключается в модификации данного кода, чтобы убедиться, что каналы, используемые в качестве параметров функций, указаны только для отправки или приема значений.

- Основы Golang
- Понимание каналов и их использования в Golang

```sh
$ go run channel-directions.go
passed message
```

Ниже представлен полный код:

```go
// При использовании каналов в качестве параметров функций можно
// указать, предназначен ли канал только для отправки или приема
// значений. Эта спецификация повышает типобезопасность
// программы.

package main

import "fmt"

// Эта функция `ping` принимает только канал для отправки
// значений. Попытка получить значение из этого канала
// вызовет ошибку компиляции.
func ping(pings chan<- string, msg string) {
	pings <- msg
}

// Функция `pong` принимает один канал для приема
// (`pings`) и второй для отправки (`pongs`).
func pong(pings <-chan string, pongs chan<- string) {
	msg := <-pings
	pongs <- msg
}

func main() {
	pings := make(chan string, 1)
	pongs := make(chan string, 1)
	ping(pings, "passed message")
	pong(pings, pongs)
	fmt.Println(<-pongs)
}

```
