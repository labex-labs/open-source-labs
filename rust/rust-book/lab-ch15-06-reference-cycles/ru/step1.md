# Reference Cycles Can Leak Memory

Гарантии безопасности памяти Rust делают сложной, но не невозможной случайную ситуацию, когда память никогда не очищается (известная как _утечка памяти_). Полное предотвращение утечек памяти не входит в число гарантий Rust, что означает, что утечки памяти являются безопасными в Rust. Мы можем увидеть, что Rust позволяет утечки памяти, используя `Rc<T>` и `RefCell<T>`: можно создать ссылки, при которых элементы ссылаются друг на друга в цикле. Это создает утечки памяти, потому что счетчик ссылок каждого элемента в цикле никогда не достигнет 0, и значения никогда не будут удалены.
