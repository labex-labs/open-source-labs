# 릴리스용 빌드

프로젝트가 마침내 릴리스 준비가 되면 `cargo build --release`를 사용하여 최적화된 상태로 컴파일할 수 있습니다.

```bash
cargo build --release
```

이 명령은 `target/debug` 대신 `target/release`에 실행 파일을 생성합니다. 최적화는 Rust 코드를 더 빠르게 실행하게 하지만, 최적화를 켜면 프로그램 컴파일 시간이 길어집니다. 이것이 개발용과 최종 프로그램을 빌드하기 위한 두 가지 프로파일이 있는 이유입니다. 개발 시에는 빠르게 자주 다시 빌드하고 싶고, 최종 프로그램은 반복적으로 다시 빌드되지 않으며 가능한 한 빠르게 실행되도록 합니다. 코드의 실행 시간을 벤치마킹하는 경우, `cargo build --release`를 실행하고 `target/release`의 실행 파일로 벤치마킹해야 합니다.
