# コンパイルと実行は別々のステップ

さっき新しく作成したプログラムを実行しましたので、このプロセスの各ステップを見てみましょう。

Rustプログラムを実行する前に、Rustコンパイラを使ってコンパイルする必要があります。`rustc`コマンドを入力し、その後にソースファイルの名前を渡します。例えばこのようになります。

```bash
rustc main.rs
```

CやC++の経験がある方は、これが`gcc`や`clang`に似ていることに気付くでしょう。正常にコンパイルされると、Rustはバイナリ実行可能ファイルを出力します。

Linux、macOS、およびWindowsのPowerShellでは、シェルで`ls`コマンドを入力することで実行可能ファイルを確認できます。

```bash
$ ls
main main.rs
```

ここから、`main`ファイルを実行します。例えばこのように。

```bash
./main
```

もし`main.rs`が「Hello, world!」プログラムであれば、この行はターミナルに「Hello, world!」を表示します。

Ruby、Python、またはJavaScriptのような動的言語に慣れている方は、コンパイルと実行を別々のステップとして行うことには馴染みがないかもしれません。Rustは事前コンパイル言語です。つまり、プログラムをコンパイルして実行可能ファイルを他人に渡すことができ、Rustをインストールしていなくても実行できます。`.rb`、`.py`、または`.js`ファイルを他人に渡した場合、それぞれRuby、Python、またはJavaScriptの実装がインストールされている必要があります。しかし、それらの言語では、プログラムをコンパイルして実行するためには1つのコマンドだけで済みます。言語設計においては、すべてトレードオフです。

単純なプログラムであれば`rustc`でコンパイルするだけで大丈夫ですが、プロジェクトが大きくなるにつれて、すべてのオプションを管理し、コードを共有しやすくするための手段が必要になります。次に、Cargoツールを紹介します。これは、現実世界のRustプログラムを書くのに役立ちます。
