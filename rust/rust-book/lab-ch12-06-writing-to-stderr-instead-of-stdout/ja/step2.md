# エラーが書き込まれる場所を確認する

まず、`minigrep` によって印刷されるコンテンツが現在どのように標準出力に書き込まれているかを確認しましょう。これには、標準出力ストリームをファイルにリダイレクトしながら意図的にエラーを発生させます。標準エラーストリームはリダイレクトしませんので、標準エラーに送信されるコンテンツは画面に表示され続けます。

コマンドラインプログラムはエラーメッセージを標準エラーストリームに送信することが期待されています。そのため、標準出力ストリームをファイルにリダイレクトしても、エラーメッセージは画面に表示され続けます。しかし、私たちのプログラムは現在適切な動作をしていません。エラーメッセージの出力が代わりにファイルに保存されてしまうことがわかります！

この動作を示すために、`>` とリダイレクトしたい標準出力ストリームのファイルパス _output.txt_ を使ってプログラムを実行します。引数は渡さず、エラーを発生させます。

```bash
cargo run > output.txt
```

`>` 構文はシェルに対して、標準出力の内容を画面ではなく _output.txt_ に書き込むように指示します。期待していたエラーメッセージが画面に表示されなかったので、それはファイルに保存されたはずです。これが _output.txt_ に含まれている内容です。

```rust
Problem parsing arguments: not enough arguments
```

確かに、私たちのエラーメッセージは標準出力に印刷されています。このようなエラーメッセージが標準エラーに印刷されるようにすると、成功した実行からのデータのみがファイルに保存されるため、はるかに便利です。そこで、これを変更します。
