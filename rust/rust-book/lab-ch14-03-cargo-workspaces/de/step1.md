# Cargo Workspaces

Im Kapitel 12 haben wir ein Paket gebaut, das einen Binärkraten und einen Bibliothekskraten enthielt. Wenn sich Ihr Projekt entwickelt, könnten Sie feststellen, dass der Bibliothekskraten immer größer wird und Sie möchten Ihr Paket weiter in mehrere Bibliothekskrate aufteilen. Cargo bietet eine Funktion namens _Workspaces_, die Ihnen helfen kann, mehrere miteinander verbundene Pakete zu verwalten, die parallel entwickelt werden.
