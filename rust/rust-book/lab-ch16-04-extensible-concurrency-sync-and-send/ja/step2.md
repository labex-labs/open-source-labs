# Send によるスレッド間での所有権の移転の許可

`Send` マーカートレイトは、`Send` を実装する型の値の所有権がスレッド間で移転できることを示します。ほとんどの Rust 型は `Send` ですが、いくつかの例外があります。それには `Rc<T>` が含まれます。これは `Send` にできないのは、`Rc<T>` 値をクローンしてクローンの所有権を別のスレッドに移そうとすると、両方のスレッドが同時に参照カウントを更新する可能性があるからです。このため、`Rc<T>` は、スレッドセーフなパフォーマンスペナルティを支払いたくない単一スレッドの状況で使用するために実装されています。

したがって、Rust の型システムとトレイト境界により、不注意に `Rc<T>` 値をスレッド間で送信することは決してできません。リスト 16-14 でこれを試したとき、`the trait `Send`is not implemented for`Rc<Mutex<i32>>` というエラーが表示されました。`Send`である`Arc<T>` に切り替えると、コードはコンパイルされました。

完全に `Send` 型で構成される型は、自動的に `Send` としてマークされます。第 19 章で説明するポインタ以外のほとんどのプリミティブ型は `Send` です。
