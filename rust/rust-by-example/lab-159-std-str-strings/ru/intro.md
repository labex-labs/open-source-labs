# Введение

В этом лабе мы будем изучать концепцию строк в Rust. Rust имеет два типа строк: `String` и `&str`.

`String` - это строка, выделенная в куче, которая может увеличиваться и гарантированно является допустимой последовательностью UTF-8. С другой стороны, `&str` - это среза, который указывает на допустимую последовательность UTF-8 и может использоваться для просмотра `String`.

В Rust литералы строк могут быть записаны различными способами, включая использование экранирования для представления специальных символов. Например, `\x3F` представляет собой знак вопроса, а `\u{211D}` представляет собой кодовые точки Unicode. Также можно использовать сырые литералы строк, если нужно записать строку как есть, без экранирования.

Если вам нужно работать с байтовыми строками, Rust предоставляет литералы байтовых строк с префиксом `b`. Байтовые строки могут иметь экранирование байтов, но не Unicode. Также можно использовать сырые байтовые строки аналогично сырым литералам строк.

Важно отметить, что `str` и `String` должны всегда быть допустимыми последовательностями UTF-8. Если вам нужно работать со строками в различных кодировках, вы можете использовать внешние crates, такие как `encoding`, для преобразования между кодировками символов.

> **Примечание:** Если в лабе не указано имя файла, вы можете использовать любое имя файла, которое хотите. Например, вы можете использовать `main.rs`, скомпилировать и запустить его с помощью `rustc main.rs &&./main`.
