# 소개

이 랩에서는 Rust 의 문자열 개념을 탐구합니다. Rust 에는 두 가지 유형의 문자열이 있습니다: `String`과 `&str`.

`String`은 힙에 할당되고, 크기가 늘어날 수 있으며, 유효한 UTF-8 시퀀스임을 보장하는 문자열입니다. 반면에, `&str`은 유효한 UTF-8 시퀀스를 가리키는 슬라이스이며, `String`을 보기 위해 사용될 수 있습니다.

Rust 에서 문자열 리터럴은 특수 문자를 표현하기 위해 이스케이프를 사용하는 것을 포함하여 다양한 방식으로 작성할 수 있습니다. 예를 들어, `\x3F`는 물음표 문자를 나타내고, `\u{211D}`는 유니코드 코드 포인트를 나타냅니다. 이스케이프 없이 문자열을 있는 그대로 작성하려면 raw 문자열 리터럴을 사용할 수도 있습니다.

바이트 문자열로 작업해야 하는 경우, Rust 는 `b` 접두사를 사용하여 바이트 문자열 리터럴을 제공합니다. 바이트 문자열은 바이트 이스케이프를 가질 수 있지만, 유니코드 이스케이프는 가질 수 없습니다. Raw 바이트 문자열도 raw 문자열 리터럴과 유사한 방식으로 사용할 수 있습니다.

`str`과 `String`은 항상 유효한 UTF-8 시퀀스여야 한다는 점에 유의하는 것이 중요합니다. 다른 인코딩으로 문자열을 사용해야 하는 경우, 문자 인코딩 간의 변환을 위해 `encoding`과 같은 외부 크레이트 (crate) 를 사용할 수 있습니다.

> **참고:** 랩에서 파일 이름을 지정하지 않은 경우, 원하는 파일 이름을 사용할 수 있습니다. 예를 들어, `main.rs`를 사용하고, `rustc main.rs && ./main`으로 컴파일하고 실행할 수 있습니다.
