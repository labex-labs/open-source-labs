# はじめに

この実験では、Rustにおける文字列の概念を探ります。Rustには2種類の文字列があります。`String` と `&str` です。

`String` は、ヒープ上に割り当てられ、拡張可能な文字列であり、有効なUTF-8シーケンスであることが保証されています。一方、`&str` は、有効なUTF-8シーケンスを指すスライスであり、`String` の中身を参照するために使用できます。

Rustでは、文字列リテラルを様々な方法で記述できます。たとえば、エスケープ文字を使って特殊文字を表すことができます。たとえば、`\x3F` は疑問符文字を表し、`\u{211D}` はUnicodeコードポイントを表します。エスケープ文字を使わずにそのままの文字列を書きたい場合は、生文字列リテラルも使うことができます。

バイト文字列として作業する必要がある場合は、Rustは `b` 接頭辞を使ってバイト文字列リテラルを提供します。バイト文字列にはバイトエスケープがありますが、Unicodeエスケープはありません。生バイト文字列も生文字列リテラルと同じように使うことができます。

重要なことは、`str` と `String` は常に有効なUTF-8シーケンスでなければならないということです。異なるエンコーディングの文字列を扱う必要がある場合は、文字エンコーディング間の変換に `encoding` のような外部クレートを使うことができます。

> **注:** 実験でファイル名が指定されていない場合は、好きなファイル名を使うことができます。たとえば、`main.rs` を使って、`rustc main.rs &&./main` でコンパイルして実行することができます。
