# はじめに

この実験では、Rust における文字列の概念を探ります。Rust には 2 種類の文字列があります。`String` と `&str` です。

`String` は、ヒープ上に割り当てられ、拡張可能な文字列であり、有効な UTF-8 シーケンスであることが保証されています。一方、`&str` は、有効な UTF-8 シーケンスを指すスライスであり、`String` の中身を参照するために使用できます。

Rust では、文字列リテラルを様々な方法で記述できます。たとえば、エスケープ文字を使って特殊文字を表すことができます。たとえば、`\x3F` は疑問符文字を表し、`\u{211D}` は Unicode コードポイントを表します。エスケープ文字を使わずにそのままの文字列を書きたい場合は、生文字列リテラルも使うことができます。

バイト文字列として作業する必要がある場合は、Rust は `b` 接頭辞を使ってバイト文字列リテラルを提供します。バイト文字列にはバイトエスケープがありますが、Unicode エスケープはありません。生バイト文字列も生文字列リテラルと同じように使うことができます。

重要なことは、`str` と `String` は常に有効な UTF-8 シーケンスでなければならないということです。異なるエンコーディングの文字列を扱う必要がある場合は、文字エンコーディング間の変換に `encoding` のような外部クレートを使うことができます。

> **注：** 実験でファイル名が指定されていない場合は、好きなファイル名を使うことができます。たとえば、`main.rs` を使って、`rustc main.rs &&./main` でコンパイルして実行することができます。
