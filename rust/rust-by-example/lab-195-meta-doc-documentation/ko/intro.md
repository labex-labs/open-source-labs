# 소개

이 실습에서는 `target/doc`에 문서를 생성하기 위해 `cargo doc`를 사용할 수 있습니다. 또한 모든 테스트, 문서 테스트 포함, 실행을 위해 `cargo test`를 사용하고, 문서 테스트만 실행하려면 `cargo test --doc`를 사용할 수 있습니다. `///`로 표시되는 문서 주석은 `rustdoc`에 의해 문서로 컴파일되며 Markdown 을 지원합니다. 이러한 주석은 대규모 프로젝트에서 코드를 문서화하는 데 유용합니다. `inline`, `no_inline`, `hidden`과 같은 문서 속성은 `rustdoc`와 자주 사용됩니다. Rustdoc 은 표준 라이브러리 문서를 포함하여 커뮤니티에서 문서 생성에 널리 사용됩니다.

> **참고:** 실습에서 파일 이름을 지정하지 않으면 원하는 파일 이름을 사용할 수 있습니다. 예를 들어 `main.rs`를 사용하고 `rustc main.rs && ./main`으로 컴파일 및 실행할 수 있습니다.
