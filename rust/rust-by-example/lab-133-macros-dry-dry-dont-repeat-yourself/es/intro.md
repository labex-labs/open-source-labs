# Introducción

En este laboratorio, exploramos el concepto de DRY (Don't Repeat Yourself) en la codificación mediante el uso de macros para extraer las partes comunes de funciones y suites de pruebas.

> **Nota:** Si el laboratorio no especifica un nombre de archivo, puede usar cualquier nombre de archivo que desee. Por ejemplo, puede usar `main.rs`, compilar y ejecutarlo con `rustc main.rs &&./main`.
