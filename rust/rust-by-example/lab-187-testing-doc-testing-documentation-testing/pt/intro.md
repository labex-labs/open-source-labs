# Introdução

Neste laboratório, a principal forma de documentar um projeto Rust é através da anotação do código-fonte com comentários de documentação, escritos na especificação CommonMark Markdown e que suportam blocos de código. O Rust garante a correção e esses blocos de código são compilados e usados como testes de documentação. Estes testes são executados automaticamente ao usar o comando `cargo test`. A motivação por trás dos testes de documentação é servir como exemplos que exercitam a funcionalidade e permitem usar exemplos da documentação como trechos de código completos.

> **Nota:** Se o laboratório não especificar um nome de arquivo, você pode usar qualquer nome de arquivo que desejar. Por exemplo, você pode usar `main.rs`, compilá-lo e executá-lo com `rustc main.rs && ./main`.
