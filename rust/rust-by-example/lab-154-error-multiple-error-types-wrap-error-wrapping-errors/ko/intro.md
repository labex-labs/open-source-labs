# 소개

이 랩에서는 사용자 정의 오류 타입으로 오류를 래핑하는 대체 접근 방식을 보여줍니다. 코드 예제는 표준 라이브러리의 `ParseIntError`를 래핑하는 `DoubleError` 열거형을 오류 변형으로 사용하는 `Result` 타입 별칭을 정의하는 방법을 보여줍니다. `fmt::Display`, `error::Error`, 및 `From` 트레이트를 구현함으로써, 사용자 정의 오류 타입은 추가 정보를 제공하고 기본 오류를 처리할 수 있습니다.

> **참고:** 랩에서 파일 이름을 지정하지 않은 경우, 원하는 파일 이름을 사용할 수 있습니다. 예를 들어, `main.rs`를 사용하고 `rustc main.rs && ./main`으로 컴파일하고 실행할 수 있습니다.
