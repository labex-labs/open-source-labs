# 소개

이 랩에서는 Rust 의 차용 (borrowing) 에 대해 배우며, 이는 값으로 객체를 전달하는 대신 ('T') 참조 ('&T') 를 사용하여 소유권을 가져가지 않고 데이터에 접근할 수 있게 해줍니다. 차용 검사기 (borrow checker) 는 참조가 항상 유효한 객체를 가리키도록 보장하며, 차용 중인 객체의 파괴를 방지합니다.

> **참고:** 랩에서 파일 이름을 지정하지 않은 경우, 원하는 파일 이름을 사용할 수 있습니다. 예를 들어, `main.rs`를 사용하고, `rustc main.rs && ./main`으로 컴파일하고 실행할 수 있습니다.
