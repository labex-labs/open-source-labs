# 소개

이 실습에서는 라이브러리의 공개 인터페이스를 사용하여 라이브러리의 여러 부분을 함께 테스트하는 통합 테스트에 대해 설명합니다. 통합 테스트는 Rust 크레이트의 `src` 디렉토리 옆에 있는 `tests` 디렉토리에 배치할 수 있으며, `cargo test` 명령을 사용하여 실행할 수 있습니다. 또한, 공개 함수를 가진 모듈을 만들고 테스트 내에서 가져오면 통합 테스트 간에 공통 코드를 공유할 수 있습니다.

> **참고:** 실습에서 파일 이름을 지정하지 않으면 원하는 파일 이름을 사용할 수 있습니다. 예를 들어 `main.rs`를 사용하고 `rustc main.rs && ./main`으로 컴파일 및 실행할 수 있습니다.
