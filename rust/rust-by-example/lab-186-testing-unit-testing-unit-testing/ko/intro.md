# 소개

이 실습에서는 Rust 의 단위 테스트에 대해 배웁니다. 단위 테스트는 설정, 코드 실행 및 결과 확인을 통해 테스트가 아닌 코드를 검증하는 Rust 함수입니다. 이러한 테스트는 `tests` 모듈에 `#[cfg(test)]` 특성과 `#[test]` 특성으로 표시되어 작성됩니다. 테스트 함수에서 예외가 발생하면 테스트가 실패할 수 있으며, `assert!`, `assert_eq!`, `assert_ne!`와 같은 도움말 매크로가 확인에 사용됩니다. Rust 2018 에서는 단위 테스트가 `Result<()>`를 반환하여 간결한 테스트를 위해 `?` 연산자를 사용할 수 있도록 허용합니다. `#[should_panic]` 특성을 사용하여 예외 발생 테스트도 지원합니다. 특정 테스트는 `cargo test` 명령어로 테스트 이름을 사용하여 실행할 수 있으며, `#[ignore]` 특성을 사용하거나 `cargo test -- --ignored`를 실행하여 테스트를 무시할 수 있습니다.

> **참고:** 실습에서 파일 이름을 지정하지 않으면 원하는 파일 이름을 사용할 수 있습니다. 예를 들어 `main.rs`를 사용하고 `rustc main.rs && ./main`으로 컴파일 및 실행할 수 있습니다.
