# 简介

在本实验中，我们将学习 Rust 中的单元测试。单元测试是 Rust 函数，通过执行设置、运行代码并断言结果来验证非测试代码。这些测试是在带有 `#[cfg(test)]` 属性的 `tests` 模块中编写的，并使用 `#[test]` 属性进行标记。如果测试函数中的某些内容导致恐慌，测试可能会失败，并且像 `assert!`、`assert_eq!` 和 `assert_ne!` 这样的辅助宏用于断言。Rust 2018 允许单元测试返回 `Result<()>` 以使用 `?` 运算符进行简洁的测试。此外，还支持使用 `#[should_panic]` 属性来测试恐慌情况。可以使用 `cargo test` 命令并加上测试名称来运行特定的测试，也可以使用 `#[ignore]` 属性或通过运行 `cargo test -- --ignored` 来忽略测试。

> **注意**：如果实验未指定文件名，你可以使用任何你想要的文件名。例如，你可以使用 `main.rs`，并通过 `rustc main.rs &&./main` 进行编译和运行。
