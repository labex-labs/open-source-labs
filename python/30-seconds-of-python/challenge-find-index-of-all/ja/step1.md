# すべての一致するインデックスを見つける

## 問題

`find_index_of_all(lst, fn)` という関数を書きなさい。この関数は、リスト `lst` とテスト関数 `fn` を引数として受け取り、`fn` が `True` を返す `lst` 内のすべての要素のインデックスのリストを返します。

### 入力

- 整数のリスト `lst`。
- 整数を入力として受け取り、ブール値を返すテスト関数 `fn`。

### 出力

- `fn` が `True` を返す `lst` 内のすべての要素のインデックスを表す整数のリスト。

## 例

```python
find_index_of_all([1, 2, 3, 4], lambda n: n % 2 == 1) # [0, 2]
find_index_of_all([1, 2, 3, 4], lambda n: n > 2) # [2, 3]
find_index_of_all([1, 2, 3, 4], lambda n: n < 0) # []
```

### 注意

- 最初の例では、テスト関数 `lambda n: n % 2 == 1` は整数が奇数であるかどうかをチェックします。この関数は `[0, 2]` を返します。なぜなら、リスト `[1, 2, 3, 4]` のインデックス `0` と `2` にある要素は奇数だからです。
- 2番目の例では、テスト関数 `lambda n: n > 2` は整数が `2` より大きいかどうかをチェックします。この関数は `[2, 3]` を返します。なぜなら、リスト `[1, 2, 3, 4]` のインデックス `2` と `3` にある要素は `2` より大きいからです。
- 3番目の例では、テスト関数 `lambda n: n < 0` は整数が負であるかどうかをチェックします。この関数は `[]` を返します。なぜなら、リスト `[1, 2, 3, 4]` には負の要素がないからです。
