# Согласование типов

В текущем классе `Stock` есть переменная класса `_types`, которая дает преобразования при чтении из файла, но также есть свойства, которые налагают ограничения на типы. Кто в этом управляет? Исправьте определения свойств так, чтобы они использовали типы, указанные в переменной класса `_types`. Убедитесь, что свойства работают, когда типы изменяются путем наследования. Например:

```python
>>> from decimal import Decimal
>>> class DStock(Stock):
        _types = (str, int, Decimal)

>>> s = DStock('AA', 50, Decimal('91.1'))
>>> s.price = 92.3
Traceback (most recent call last):
...
TypeError: Expected a Decimal
>>>
```

**Обсуждение**

Результрующий класс `Stock` в конце этого лабораторного занятия представляет собой запутанный комбинацию свойств, проверки типов, конструкторов и других деталей. Представьте, насколько неприятно поддерживать код, в котором есть десятки или сотни таких определений классов.

Мы собираемся понять, как значительно упростить вещи, но это займет некоторое время и более продвинутые техники. Будьте в курсе.
