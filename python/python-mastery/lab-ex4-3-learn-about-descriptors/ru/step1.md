# Дескрипторы в действии

Ранее вы создали класс `Stock`, который использовал слоты, свойства и другие особенности. Все эти особенности реализуются с использованием протокола дескрипторов. Посмотрите, как это работает, выполнив этот простой эксперимент.

Во - первых, создайте объект акции и попытайтесь получить доступ к нескольким атрибутам:

```python
>>> s = Stock('GOOG', 100, 490.10)
>>> s.name
'GOOG'
>>> s.shares
100
>>>
```

Теперь обратите внимание, что эти атрибуты находятся в словаре класса.

```python
>>> Stock.__dict__.keys()
['sell', '__module__', '__weakref__', 'price', '_price','shares', '_shares',
'__slots__', 'cost', '__repr__', '__doc__', '__init__']
>>>
```

Попробуйте следующие шаги, которые иллюстрируют, как дескрипторы получают и назначают значения для экземпляра:

```python
>>> q = Stock.__dict__['shares']
>>> q.__get__(s)
100
>>> q.__set__(s,75)
>>> s.shares
75
>>> q.__set__(s, '75')
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
  File "stock.py", line 23, in shares
    raise TypeError('Expected an integer')
TypeError: Expected an integer
>>>
```

Вызов `__get__()` и `__set__()` происходит автоматически при любом доступе к экземплярам.
