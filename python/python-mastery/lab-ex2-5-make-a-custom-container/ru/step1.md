# Рост списка

Списки в Python оптимизированы для выполнения операций `append()`. Каждый раз, когда список растет, он выделяет более крупный кусок памяти, чем ему на самом деле нужно, с тем ожиданием, что позже в список будут добавлены больше данных. Если добавляются новые элементы и есть свободное место, операция `append()` сохраняет элемент без выделения дополнительной памяти.

Проверьте эту особенность списков, используя функцию `sys.getsizeof()` для списка и добавив несколько элементов.

```python
>>> import sys
>>> items = []
>>> sys.getsizeof(items)
64
>>> items.append(1)
>>> sys.getsizeof(items)
96
>>> items.append(2)
>>> sys.getsizeof(items)    # Заметьте, как размер не увеличивается
96
>>> items.append(3)
>>> sys.getsizeof(items)    # Здесь также не увеличивается
96
>>> items.append(4)
>>> sys.getsizeof(items)    # Еще нет.
96
>>> items.append(5)
>>> sys.getsizeof(items)    # Заметьте, что размер резко увеличился
128
>>>
```

Список хранит свои элементы по ссылке. Таким образом, память, требуемая для каждого элемента, представляет собой единственный адрес памяти. На 64-разрядной машине адрес обычно составляет 8 байт. Однако, если Python скомпилирован для 32-разрядной системы, он может быть 4 байта, и числа в приведенном выше примере будут в два раза меньше.
