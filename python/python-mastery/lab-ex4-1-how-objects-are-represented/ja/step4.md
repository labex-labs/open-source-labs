# クラスの役割

クラス定義を構成する定義は、そのクラスのすべてのインスタンスによって共有されます。すべてのインスタンスが関連付けられたクラスに戻るリンクがあることに注意してください。

```python
>>> goog.__class__
... 出力を見てください...
>>> ibm.__class__
... 出力を見てください...
>>>
```

インスタンスでメソッドを呼び出してみましょう。

```python
>>> goog.cost()
49010.0
>>> ibm.cost()
4561.5
>>>
```

`cost` という名前が `goog.__dict__` または `ibm.__dict__` のどちらにも定義されていないことに注意してください。代わりに、それはクラス辞書によって提供されています。これを試してみましょう。

```python
>>> SimpleStock.__dict__['cost']
... 出力を見てください...
>>>
```

辞書を介して直接 `cost()` メソッドを呼び出してみましょう。

```python
>>> SimpleStock.__dict__['cost'](goog)
49010.00
>>> SimpleStock.__dict__['cost'](ibm)
4561.5
>>>
```

クラス定義で定義された関数をどのように呼び出しているかと、`self` 引数がインスタンスをどのように取得するかに注意してください。

クラスに新しい値を追加すると、それはすべてのインスタンスに対して表示されるクラス変数になります。試してみましょう。

```python
>>> SimpleStock.spam = 42
>>> ibm.spam
42
>>> goog.spam
42
>>>
```

`spam` がインスタンス辞書の一部ではないことに注意してください。

```python
>>> ibm.__dict__
... 出力を見てください...
>>>
```

代わりに、それはクラス辞書の一部です。

```python
>>> SimpleStock.__dict__['spam']
42
>>>
```

基本的に、これがクラスの本質です。つまり、インスタンスによって共有される値のコレクションです。
