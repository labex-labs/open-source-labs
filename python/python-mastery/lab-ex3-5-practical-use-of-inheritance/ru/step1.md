# Подготовка

Одним из основных применений классов в Python является написание кода, который может быть расширен/адаптирован различными способами. Чтобы проиллюстрировать это, в упражнении 3.2 вы создали функцию `print_table()`, которая создаёт таблицы. Вы использовали её для вывода данных из списка `portfolio`. Например:

```python
>>> import stock
>>> import reader
>>> import tableformat
>>> portfolio = reader.read_csv_as_instances('portfolio.csv', stock.Stock)
>>> tableformat.print_table(portfolio, ['name','shares','price'])
      name     shares      price
---------- ---------- ----------
        AA        100       32.2
       IBM         50       91.1
       CAT        150      83.44
      MSFT        200      51.23
        GE         95      40.37
      MSFT         50       65.1
       IBM        100      70.44
>>>
```

Предположим, что вы хотите, чтобы функция `print_table()` могла создавать таблицы в любом количестве форматов вывода, таких как CSV, XML, HTML, Excel и т.д. Попытка модифицировать функцию для одновременной поддержки всех этих форматов вывода будет болезненной. Лучший способ сделать это — перенести код форматирования, связанный с выводом, в класс и использовать наследование для реализации различных форматов вывода.
