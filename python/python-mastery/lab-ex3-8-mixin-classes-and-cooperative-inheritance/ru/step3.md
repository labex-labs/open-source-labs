# Сделание его разумным

Использование миксинов может быть полезным инструментом для разработчиков фреймворков для сокращения количества кода, который необходимо писать. Однако заставлять пользователей помнить, как правильно составлять классы и использовать множественное наследование, может привести их в ступор. В упражнении 3.5 вы написали функцию `create_formatter()`, которая упрощала создание пользовательского форматтера. Возьмите эту функцию и расширьте ее, чтобы она понимала несколько необязательных аргументов, связанных с классами-миксинами. Например:

```python
>>> from tableformat import create_formatter
>>> formatter = create_formatter('csv', column_formats=['"%s"','%d','%0.2f'])
>>> print_table(portfolio, ['name','shares','price'], formatter)
name,shares,price
"AA",100,32.20
"IBM",50,91.10
"CAT",150,83.44
"MSFT",200,51.23
"GE",95,40.37
"MSFT",50,65.10
"IBM",100,70.44

>>> formatter = create_formatter('text', upper_headers=True)
>>> print_table(portfolio, ['name','shares','price'], formatter)
      NAME     SHARES      PRICE
---------- ---------- ----------
        AA        100       32.2
       IBM         50       91.1
       CAT        150      83.44
      MSFT        200      51.23
        GE         95      40.37
      MSFT         50       65.1
       IBM        100      70.44
>>>
```

Под капотом функция `create_formatter()` правильно составит классы и вернет правильный экземпляр `TableFormatter`.
