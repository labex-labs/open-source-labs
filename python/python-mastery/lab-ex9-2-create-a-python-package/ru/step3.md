# Исправление инструкций импорта

Теперь давайте разберемся, почему нам это нужно сделать. Когда мы переместили наши файлы в пакет `structly`, способ, которым Python ищет модули, изменился. Инструкции импорта в каждом файле должны быть обновлены, чтобы соответствовать новой структуре пакета. Это важно, так как Python использует эти инструкции импорта для поиска и использования кода из других модулей.

Файл `structure.py` особенно важен для обновления. Он импортирует функции и классы из файла `validate.py`. Поскольку оба этих файла теперь находятся в одном пакете `structly`, мы должны соответствующим образом скорректировать инструкцию импорта.

Начнем с открытия файла `structly/structure.py` в редакторе. Вы можете либо кликнуть на `structly/structure.py` в проводнике файлов, либо выполнить следующую команду в терминале:

```bash
# Click on structly/structure.py in the file explorer or run:
code structly/structure.py
```

После открытия файла посмотрите на первую строку инструкции импорта. В настоящее время она выглядит так:

```python
from validate import validate_type, PositiveInteger, PositiveFloat, String
```

Эта инструкция была правильной, когда файлы имели другую структуру. Но теперь нам нужно изменить ее, чтобы сообщить Python искать модуль `validate` в том же пакете. Поэтому мы меняем ее на:

```python
from .validate import validate_type, PositiveInteger, PositiveFloat, String
```

Точка (`.`) перед `validate` здесь является ключевым элементом. Это специальный синтаксис в Python, называемый относительным импортом. Он сообщает Python искать модуль `validate` в том же пакете, что и текущий модуль (в данном случае это `structure.py`).

После внесения этого изменения обязательно сохраните файл. Сохранение важно, так как оно делает изменения постоянными, и Python будет использовать обновленную инструкцию импорта при запуске вашего кода.

Теперь проверим наши другие файлы, чтобы узнать, нуждаются ли они в обновлении.

1. `structly/reader.py` - Этот файл не импортирует ни из одного из наших пользовательских модулей. Это означает, что нам не нужно вносить никаких изменений в него.
2. `structly/tableformat.py` - Как и файл `reader.py`, этот файл также не импортирует ни из одного из наших пользовательских модулей. Поэтому здесь также не требуются изменения.
3. `structly/validate.py` - Как и предыдущие два файла, он не импортирует ни из одного из наших пользовательских модулей. Следовательно, нам не нужно его изменять.

В реальном программировании ваши проекты могут иметь более сложные отношения между модулями. Когда вы перемещаете файлы для создания или изменения структуры пакета, всегда помните обновлять инструкции импорта. Это гарантирует, что ваш код сможет правильно найти и использовать необходимые модули.
