# Закрытие генератора

Одним из распространенных вопросов, связанных с генераторами, является их срок жизни и сбор мусора. Например, генератор `follow()` бесконечно работает в бесконечном цикле `while`. Что произойдет, если цикл итерации, который его запускает, остановится? Кроме того, есть ли способ принудительно прервать генератор?

Измените функцию `follow()`, чтобы весь код был помещен в блок `try-except` следующим образом:

```python
def follow(filename):
    try:
        with open(filename,'r') as f:
            f.seek(0,os.SEEK_END)
            while True:
                 line = f.readline()
                 if line == '':
                     time.sleep(0.1)    # Sleep briefly to avoid busy wait
                     continue
                 yield line
    except GeneratorExit:
        print('Following Done')
```

Теперь проведите несколько экспериментов:

```python
>>> from follow import follow
>>> # Experiment: Garbage collection of a running generator
>>> f = follow('stocklog.csv')
>>> next(f)
'"MO",70.29,"6/11/2007","09:30.09",-0.01,70.25,70.30,70.29,365314\n'
>>> del f
Following Done
>>> # Experiment: Closing a generator
>>> f = follow('stocklog.csv')
>>> for line in f:
        print(line,end='')
        if 'IBM' in line:
            f.close()

"VZ",42.91,"6/11/2007","09:34.28",-0.16,42.95,42.91,42.78,210151
"HPQ",45.76,"6/11/2007","09:34.29",0.06,45.80,45.76,45.59,257169
"GM",31.45,"6/11/2007","09:34.31",0.45,31.00,31.50,31.45,582429
...
"IBM",102.86,"6/11/2007","09:34.44",-0.21,102.87,102.86,102.77,147550
Following Done
>>> for line in f:
        print(line, end='')    # No output: generator is done

>>>
```

В этих экспериментах можно увидеть, что исключение `GeneratorExit` возникает, когда генератор собирается в мусор или явно закрывается с помощью его метода `close()`.

Одним из дополнительных направлений исследования является возможность продолжить итерацию по генератору, если вы выйдете из цикла `for`. Например, попробуйте это:

```python
>>> f = follow('stocklog.csv')
>>> for line in f:
        print(line,end='')
        if 'IBM' in line:
            break

"CAT",78.36,"6/11/2007","09:37.19",-0.16,78.32,78.36,77.99,237714
"VZ",42.99,"6/11/2007","09:37.20",-0.08,42.95,42.99,42.78,268459
...
"IBM",102.91,"6/11/2007","09:37.31",-0.16,102.87,102.91,102.77,190859
>>> # Resume iteration
>>> for line in f:
        print(line,end='')
        if 'IBM' in line:
            break

"AA",39.58,"6/11/2007","09:39.28",-0.08,39.67,39.58,39.31,243159
"HPQ",45.94,"6/11/2007","09:39.29",0.24,45.80,45.94,45.59,408919
...
"IBM",102.95,"6/11/2007","09:39.44",-0.12,102.87,102.95,102.77,225350
>>> del f
Following Done
>>>
```

В целом, вы можете выйти из текущей итерации и продолжить ее позже, если необходимо. Вам просто нужно убедиться, что объект генератора не будет принудительно закрыт или собран в мусор каким-либо образом.
