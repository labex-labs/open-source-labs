# Генераторные выражения и функции редукции

Генераторные выражения особенно полезны для подачи данных в функции, такие как `sum()`, `min()`, `max()`, `any()` и т.д. Попробуйте несколько примеров, используя данные портфеля из предыдущего примера. Обратите внимание, что в этих примерах отсутствуют дополнительные квадратные скобки ([]), которые появлялись при использовании списочных включений.

```python
>>> from readport import read_portfolio
>>> portfolio = read_portfolio('portfolio.csv')
>>> sum(s['shares']*s['price'] for s in portfolio)
44671.15
>>> min(s['shares'] for s in portfolio)
50
>>> any(s['name'] == 'IBM' for s in portfolio)
True
>>> all(s['name'] == 'IBM' for s in portfolio)
False
>>> sum(s['shares'] for s in portfolio if s['name'] == 'IBM')
150
>>>
```

Вот неожиданное использование генераторного выражения для создания значений, разделенных запятыми:

```python
>>> s = ('GOOG',100,490.10)
>>> ','.join(s)
... обратите внимание, что это не работает...
>>> ','.join(str(x) for x in s)    # Это работает
'GOOG,100,490.1'
>>>
```

Синтаксис в примерах выше требует некоторого времени для привычки, но важный момент заключается в том, что ни одна из операций не создает полностью заполненный список результатов. Это позволяет экономить значительную память. Однако, нужно убедиться, что не переборщить с синтаксисом.
