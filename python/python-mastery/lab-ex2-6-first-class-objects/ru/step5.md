# Глубокое мышление

В этом упражнении вы написали две функции: `read_csv_as_dicts()` и `read_csv_as_columns()`. Эти функции представляют данные пользователю одинаково. Например:

```python
>>> data1 = read_csv_as_dicts('ctabus.csv', [str, str, str, int])
>>> len(data1)
577563
>>> data1[0]
{'route': '3', 'date': '01/01/2001', 'daytype': 'U', 'rides': 7354}
>>> data1[1]
{'route': '4', 'date': '01/01/2001', 'daytype': 'U', 'rides': 9288}
>>>

>>> data2 = read_csv_as_columns('ctabus.csv', [str, str, str, int])
>>> len(data2)
577563
>>> data2[0]
{'route': '3', 'date': '01/01/2001', 'daytype': 'U', 'rides': 7354}
>>> data2[1]
{'route': '4', 'date': '01/01/2001', 'daytype': 'U', 'rides': 9288}
>>>
```

Фактически, вы можете использовать любую из этих функций в коде анализа данных CTA, который вы написали. Тем не менее, под капотом происходят совершенно разные вещи. Функция `read_csv_as_columns()` хранит данные в другом представлении. Она relies на протоколы последовательностей Python (магические методы), чтобы представить информацию вам более полезным способом.

Это на самом деле часть гораздо большей концепции программирования - "абстракции данных". Когда вы пишете программы, то способ, которым данные представляются, часто имеет более важное значение, чем то, как данные на самом деле собираются под капотом. Хотя мы представляем данные в виде последовательности словарей, есть значительная гибкость в том, как это происходит на самом деле幕后. Это мощная идея и что-то, о чем стоит задуматься, когда вы пишете свои собственные программы.
