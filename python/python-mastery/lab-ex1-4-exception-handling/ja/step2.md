# エラーハンドリングの追加

データを処理するプログラムを書く際、不適切なデータ（形式が悪い、フィールドが欠けているなど）に関連するエラーに遭遇することがよくあります。`pcost.py` プログラムを修正して、データファイル `portfolio3.dat` を読み取り、実行してみましょう（ヒント：クラッシュするはずです）。

関数を少し修正して、不適切なデータの行から回復できるようにします。たとえば、変換関数 `int()` と `float()` は、入力を変換できない場合に `ValueError` 例外を発生させます。`try` と `except` を使って、解析できない行に関する警告メッセージをキャッチして表示します。たとえば：

```shell
解析できません: 'C - 53.08\n'
理由: 基数10のint()に対する無効なリテラル: '-'
解析できません: 'DIS - 34.20\n'
理由: 基数10のint()に対する無効なリテラル: '-'
...
```

再度、`portfolio3.dat` ファイルでプログラムを実行してみましょう。警告メッセージが表示されても、正常に実行されるはずです。
