# Поиск в повернутом массиве

## Задача

Дан отсортированный массив, который был повернут несколько раз. Нам нужно найти определенный элемент в массиве. Например, если исходный отсортированный массив был [1, 2, 3, 4, 5], а он был повернут дважды и стал [3, 4, 5, 1, 2], нам нужно найти индекс определенного элемента, такого как 4.

## Требования

Для решения этой задачи нужно учитывать следующие требования:

- Вход - это массив целых чисел.
- Мы не знаем, сколько раз массив был повернут.
- Массив был изначально отсортирован по возрастанию.
- Для вывода нужно вернуть индекс элемента, который мы ищем.
- Мы не можем предполагать, что входные данные валидны.
- Мы можем предположить, что решение помещается в память.

## Пример использования

Вот несколько примеров, как можно использовать эту функцию:

- Если не предоставляются входные данные, должно быть вызвано исключение.
- Если предоставляется пустой массив, должны быть возвращены None.
- Если элемент, который мы ищем, не найден в массиве, должны быть возвращены None.
- Если массив содержит дубликаты, функция должна по-прежнему быть able найти правильный индекс.
- Если массив не содержит дубликатов, функция должна по-прежнему быть able найти правильный индекс.
