# 回転された配列の検索

## 問題

何度か回転されたソート済み配列が与えられたとき、配列内の特定の要素を見つける必要があります。たとえば、元のソート済み配列が [1, 2, 3, 4, 5] で、2回回転されて [3, 4, 5, 1, 2] になった場合、4などの特定の要素のインデックスを見つける必要があります。

## 要件

この問題を解くには、次の要件を考慮する必要があります。

- 入力は整数の配列です。
- 配列が何度回転されたかはわかりません。
- 配列は元々昇順にソートされていました。
- 出力として、検索対象の要素のインデックスを返す必要があります。
- 入力が妥当であることを前提としてはいけません。
- 解がメモリに収まることを前提として良いです。

## 例の使い方

この関数の使い方の例をいくつか示します。

- 入力が提供されない場合、例外を発生させる必要があります。
- 空の配列が提供された場合、None を返す必要があります。
- 検索対象の要素が配列に見つからない場合、None を返す必要があります。
- 配列に重複する要素が含まれている場合でも、関数は正しいインデックスを見つけることができる必要があります。
- 配列に重複する要素が含まれていない場合でも、関数は正しいインデックスを見つけることができる必要があります。
