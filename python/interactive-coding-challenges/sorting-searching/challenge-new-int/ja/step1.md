# 欠けている整数を見つける

## 問題

32 個の非負整数の配列が与えられたとき、入力配列に存在しない整数を見つけます。解決策は最小限のメモリを使用する必要があります。

## 要件

この問題を解くには、次の要件を考慮する必要があります。

- 入力配列は非負整数を含んでいます。
- 整数の範囲は指定されていませんが、40 億個の整数に対するアプローチについて議論する必要があります。
- 32 個の整数の配列に対する解決策を実装する必要があります。
- 入力配列が有効であるとは仮定できません。

## 例の使用法

関数がどのように動作するかのいくつかの例を示します。

- 入力が None または空の配列の場合、関数は例外を発生させる必要があります。
- 入力配列から除外された整数がある場合、関数はその整数を返す必要があります。
- 入力配列から除外された整数がない場合、関数は None を返す必要があります。
