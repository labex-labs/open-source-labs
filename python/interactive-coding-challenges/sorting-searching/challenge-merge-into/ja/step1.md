# マージイントゥ

## 問題

ソート済みの2つの配列AとBが与えられたとき、Bをソート済みの順序でAにマージします。配列には重複する要素が含まれていてもよく、入力が有効でなくてもよいです。入力にはまた、AとBの実際のサイズも含まれており、これがメモリに収まると仮定できます。

この問題を解くには、AがBに十分な空間を持っているかどうか、および入力に重複する配列要素があるかどうかを考慮する必要があります。AがBに十分な空間を持っていない場合、追加のメモリを割り当てる必要があるかもしれません。入力に重複する配列要素がある場合、マージ中にこれらの重複要素が正しく処理されるようにする必要があります。

## 要件

この問題を解くには、次の要件を満たす必要があります。

- AがBに十分な空間を持っていることを確認する
- 重複する配列要素を正しく処理する
- 入力が有効であることを確認する
- 入力にAとBの実際のサイズを含める
- 入力がメモリに収まると仮定する

## 例の使用法

この問題をどのように解くかを示すために、次の例を考えてみましょう。

- AまたはBがNoneの場合、例外を発生させる必要があります。
- AまたはBの最後の要素のインデックスが0未満の場合、例外を発生させる必要があります。
- AまたはBが空の場合、結果はそれぞれAまたはBになります。
- 一般的な場合、BをAにマージする方法は次のとおりです。

```
A = [1, 3, 5, 7, 9, None, None, None]
B = [4, 5, 6]
A = [1, 3, 4, 5, 5, 6, 7, 9]
```
