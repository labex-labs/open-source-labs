# Приоритетная очередь

## Задача

Реализовать приоритетную очередь, основанную на массиве. Приоритетная очередь должна поддерживать следующие методы:

- `insert`: вставить новый элемент в приоритетную очередь
- `extract_min`: удалить и вернуть минимальный элемент из приоритетной очереди
- `decrease_key`: уменьшить ключ заданного элемента в приоритетной очереди

## Требования

Для реализации приоритетной очереди необходимо удовлетворить следующим требованиям:

- Методы, поддерживаемые приоритетной очередью, должны быть `insert`, `extract_min` и `decrease_key`.
- В приоритетной очереди не будет дублирующихся ключей.
- Не нужно валидировать входные данные.
- Приоритетная очередь должна помещаться в память.

## Пример использования

Вот несколько примеров использования методов приоритетной очереди:

### insert

- `insert` в общем случае: вставить новый узел в приоритетную очередь.

### extract_min

- `extract_min` из пустого списка: вернуть `None`.
- `extract_min` в общем случае: удалить и вернуть минимальный узел из приоритетной очереди.

### decrease_key

- `decrease_key` для недействительного ключа: вернуть `None`.
- `decrease_key` в общем случае: уменьшить ключ заданного узла в приоритетной очереди.
