# Хеш-карта

## Задача

Реализовать хеш-таблицу с методами set, get и remove. Хеш-таблица должна использовать цепочку для разрешения коллизий. Ключи могут быть только целыми числами. Мы не должны беспокоиться о коэффициентах загрузки или проверять входные данные. Можно предположить, что хеш-таблица помещается в память.

## Требования

- Ключи могут быть только целыми числами.
- Для разрешения коллизий используется цепочка.
- Коэффициенты загрузки не нужно учитывать.
- Входные данные не нужно проверять.
- Хеш-таблица помещается в память.

## Пример использования

- Метод `get`:
  - Если нет совпадающего ключа, генерируется исключение KeyError.
  - Если есть совпадающий ключ, возвращается соответствующее значение.
- Метод `set`:
  - Если нет совпадающего ключа, новая пара ключ-значение добавляется в хеш-таблицу.
  - Если есть совпадающий ключ, соответствующее значение обновляется.
- Метод `remove`:
  - Если нет совпадающего ключа, генерируется исключение KeyError.
  - Если есть совпадающий ключ, соответствующая пара ключ-значение удаляется из хеш-таблицы.
