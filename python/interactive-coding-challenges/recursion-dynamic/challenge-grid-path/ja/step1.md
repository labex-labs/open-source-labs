# グリッドパス

## 問題

有効なセルと無効なセルがある矩形グリッドが与えられたとき、ロボットが左上隅から右下隅まで移動するための有効なパスを見つける関数を実装します。有効なパスがない場合は、None を返します。入力が無効または行列が空の場合は、None を返します。

## 要件

このアルゴリズムの要件は次のとおりです。

- ロボットは右と下だけを移動できます。
- 一部のセルは通行禁止となっている場合があります。
- グリッドは矩形で、ジャグ状ではありません。
- ロボットが右下隅に到達するための有効なパスが必ずしも存在するとは限りません。
- 入力が必ずしも有効であるとは限りません。
- アルゴリズムはメモリ制約内に収まる必要があります。

## 例の使用法

次のグリッドを考えてみましょう。

```txt
o = 有効なセル
x = 無効なセル

   0  1  2  3
0  o  o  o  o
1  o  x  o  o
2  o  o  x  o
3  x  o  o  o
4  o  o  x  o
5  o  o  o  x
6  o  x  o  x
7  o  x  o  o
```

- 一般的なケース：

```txt
expected = [(0, 0), (1, 0), (2, 0),
            (2, 1), (3, 1), (4, 1),
            (5, 1), (5, 2), (6, 2),
            (7, 2), (7, 3)]
```

- 有効なパスなし：上記の例では、7 行 2 列も無効 -> None
- 入力なし -> None
- 空の行列 -> None
