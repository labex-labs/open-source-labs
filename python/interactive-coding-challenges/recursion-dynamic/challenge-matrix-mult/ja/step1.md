# 行列積

## 問題

2x2行列のリストが与えられた場合、それらを一緒に乗算する最小コストを見つける必要があります。2つの行列を乗算するコストは、必要なスカラー乗算の数です。たとえば、行列A、B、Cがあり、ABCの積を計算したい場合、コストは、結果となる行列の各要素を計算するために必要なスカラー乗算の数になります。

この問題を解くには、行列を乗算する最適な順序を見つける必要があります。行列を乗算する順序は、乗算の合計コストに影響を与えます。たとえば、行列A、B、Cがあり、ABCの積を計算したい場合、(AB)CまたはA(BC)のどちらかを計算できます。これら2つの計算のコストは異なる場合があり、コストを最小化する最適な順序を見つける必要があります。

## 要件

この問題を解くには、次の要件を考慮する必要があります。

- 行列乗算のコストのみを計算し、実際の演算順序をリスト表示する必要はありません。
- 入力が有効であることを前提としてはならず、無効な入力を処理する必要があります。
- 問題がメモリに収まることを想定できます。

## 例の使用法

この関数がどのように動作するかの例をいくつか示します。

- `None` -> `例外`
- `[]` -> `0`
- `[Matrix(2, 3), Matrix(3, 6), Matrix(6, 4), Matrix(4, 5)]` -> `124`
