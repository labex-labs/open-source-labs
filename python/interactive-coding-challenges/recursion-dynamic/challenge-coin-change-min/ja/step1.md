# コインの最小枚数での支払い

## 問題

nセント未満の額面のコインのセットが与えられたとき、これらのコインを使ってnセントを支払う方法の最小数を求める必要があります。コインは任意の組み合わせで使うことができ、各額面のコインは無限にあります。最小数を表すコインの組み合わせを報告する必要はありません。

## 要件

この問題を解くには、以下の要件を考慮する必要があります。

- コインは正確にnセントになる必要があります。
- nセントを支払うために無限にコインがあると仮定できます。
- 最小数を表すコインの組み合わせを報告する必要はありません。
- コインの額面がソートされた順序で与えられているとは仮定できません。
- これがメモリに収まると仮定できます。

## 例の使い方

この関数の使い方の例をいくつか示します。

- コイン: なし または n: なし -> 例外
- コイン: [] または n: 0 -> 0
- コイン: [1, 2, 3] または [3, 2, 1] -> 2
