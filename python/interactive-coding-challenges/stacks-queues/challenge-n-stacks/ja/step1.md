# n 個のスタック

## 問題

単一の配列を使用して n 個のスタックを実装します。スタックと配列は固定サイズで、等しいサイズになります。満杯のスタックに要素を追加すると例外が発生し、空のスタックから要素を削除すると例外が発生します。ユーザーが渡したスタックインデックスが有効で、メモリに収まることを仮定して構いません。

## 要件

単一の配列を使用して n 個のスタックを実装するには、以下の要件を満たす必要があります。

- スタックと配列は固定サイズです。
- スタックは等しいサイズです。
- 満杯のスタックに要素を追加すると例外が発生します。
- 空のスタックから要素を削除すると例外が発生します。
- ユーザーが渡したスタックインデックスが有効であると仮定できます。
- これがメモリに収まると仮定できます。

## 例の使用法

単一の配列を使用した n 個のスタックの実装をテストするには、3 つのスタックに対して以下の操作を行うことができます。

- 満杯のスタックに要素を追加 -> 例外
- 満杯でないスタックに要素を追加
- 空のスタックから要素を削除 -> 例外
- 空でないスタックから要素を削除
