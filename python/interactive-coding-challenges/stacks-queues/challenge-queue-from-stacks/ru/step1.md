# Очередь из стеков

## Задача

Реализация очереди с использованием двух стеков может быть сложной задачей. Основная идея заключается в использовании одного стека для операций добавления элементов в очередь (enqueue) и другого стека для операций удаления элементов из очереди (dequeue). Когда элемент добавляется в очередь, он помещается на первый стек. Когда элемент удаляется из очереди, он извлекается со второго стека. Если второй стек пуст, мы извлекаем все элементы из первого стека и помещаем их на второй стек в обратном порядке. Это гарантирует, что первый элемент, добавленный в очередь, будет первым, который будет удален из очереди.

## Требования

Для решения этой задачи необходимо учесть следующие требования:

- Мы должны реализовать два метода: enqueue и dequeue.
- Мы предполагаем, что у нас уже есть класс стека, который можно использовать для этой задачи.
- Мы не можем помещать значение None на стек.
- Можно предположить, что данная задача помещается в память.

## Пример использования

Вот несколько примеров того, как можно использовать нашу реализацию очереди с использованием двух стеков:

- Добавление и удаление элементов из пустого стека: Мы можем добавить элемент в очередь и затем удалить его, чтобы убедиться, что очередь работает правильно.
- Добавление и удаление элементов из непустого стека: Мы можем добавить несколько элементов в очередь и затем удалить их, чтобы убедиться, что очередь работает правильно.
- Несколько последовательных добавлений элементов: Мы можем последовательно добавить несколько элементов в очередь и затем удалить их, чтобы убедиться, что очередь работает правильно.
- Несколько последовательных удалений элементов: Мы можем добавить несколько элементов в очередь и затем последовательно удалить их, чтобы убедиться, что очередь работает правильно.
- Добавление элемента после удаления: Мы можем добавить элемент в очередь, удалить его, а затем добавить еще один элемент в очередь, чтобы убедиться, что очередь работает правильно.
- Удаление элемента после добавления: Мы можем добавить элемент в очередь, удалить его, а затем добавить еще один элемент в очередь, чтобы убедиться, что очередь работает правильно.
