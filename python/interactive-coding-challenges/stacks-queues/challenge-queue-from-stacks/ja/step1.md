# スタックからのキュー

## 問題

2 つのスタックを使ってキューを実装することは、難しい問題かもしれません。基本的な考え方は、エンキュー操作に 1 つのスタックを使い、デキュー操作にもう 1 つのスタックを使うことです。要素がエンキューされると、最初のスタックにプッシュされます。要素がデキューされると、2 番目のスタックからポップされます。2 番目のスタックが空の場合、最初のスタックからすべての要素をポップし、逆順に 2 番目のスタックにプッシュします。これにより、エンキューされた最初の要素がデキューされる最初の要素となります。

## 要件

この問題を解くには、以下の要件を考慮する必要があります。

- 2 つのメソッドを実装する必要があります。エンキューとデキュー。
- この問題に使用できるスタッククラスが既にあると仮定します。
- スタックに None 値をプッシュしてはいけません。
- この問題はメモリに収まると仮定できます。

## 例の使い方

2 つのスタックを使ったキューの実装をどのように使うかの例をいくつか示します。

- 空のスタックでのエンキューとデキュー：キューに要素をエンキューし、その後デキューして、キューが正しく動作していることを確認できます。
- 空でないスタックでのエンキューとデキュー：キューに複数の要素をエンキューし、その後デキューして、キューが正しく動作していることを確認できます。
- 連続した複数のエンキュー：キューに複数の要素を連続してエンキューし、その後デキューして、キューが正しく動作していることを確認できます。
- 連続した複数のデキュー：キューに複数の要素をエンキューし、その後連続してデキューして、キューが正しく動作していることを確認できます。
- デキューの後のエンキュー：キューに要素をエンキューし、デキューし、その後別の要素をエンキューして、キューが正しく動作していることを確認できます。
- エンキューの後のデキュー：キューに要素をエンキューし、デキューし、その後別の要素をエンキューして、キューが正しく動作していることを確認できます。
