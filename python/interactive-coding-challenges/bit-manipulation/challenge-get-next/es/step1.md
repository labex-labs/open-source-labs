# Obtener el siguiente

## Problema

Dado un entero positivo, debes encontrar el número más grande y el número más pequeño siguientes con la misma cantidad de 1's que el número dado. Por ejemplo, si la entrada es 0000 0000 1101 0111, la salida debería ser 0000 0000 1101 1011 para el número más grande siguiente y 0000 0000 1100 1111 para el número más pequeño siguiente.

## Requisitos

Para resolver este desafío, debes cumplir con los siguientes requisitos:

- La salida debe ser un entero positivo.
- Las entradas pueden no ser válidas, por lo que debes manejar las excepciones.
- La solución debe caber en memoria.

## Uso de ejemplo

A continuación se presentan algunos ejemplos de cómo usar esta función:

- Ninguno -> Excepción
- 0 -> Excepción
- Entero negativo -> Excepción
- Caso general:

```txt
    * Entrada:         0000 0000 1101 0111
    * Siguiente más grande:  0000 0000 1101 1011
    * Siguiente más pequeño: 0000 0000 1100 1111
```
