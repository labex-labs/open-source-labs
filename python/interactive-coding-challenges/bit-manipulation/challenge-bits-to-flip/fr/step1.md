# Bits To Flip

## Problème

Étant donné deux entiers A et B, déterminer le nombre de bits à inverser pour convertir A en B. Par exemple, si A = 11101 et B = 01111, le nombre de bits à inverser est 2.

## Exigences

Pour résoudre ce problème, nous devons prendre en compte les exigences suivantes :

- A et B sont toujours des entiers.
- La sortie est toujours un entier.
- A et B ont toujours le même nombre de bits.
- Les entrées peuvent ne pas être valides (None).
- La solution doit tenir en mémoire.

## Utilisation de l'exemple

Voici quelques exemples d'utilisation de cette fonction :

- Si A ou B est None, une exception doit être levée.
- Si A = 11101 et B = 01111, le nombre de bits à inverser est 2.
