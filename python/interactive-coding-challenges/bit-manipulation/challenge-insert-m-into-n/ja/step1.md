# N に M を挿入する

## 問題

2 つの 16 ビット数 `n` と `m` と 2 つのインデックス `i` と `j` が与えられたとき、`m` を `n` に挿入して、`m` がビット `j` から始まり、ビット `i` で終わるようにします。プログラムは以下のケースに対応する必要があります。

- 入力として何も与えられない場合、例外を発生させる。
- `i` または `j` に負のインデックスが与えられた場合、例外を発生させる。
- 入力が無効な場合、例外を発生させる。
- `i` から `j` までに `m` を格納するスペースが足りない場合、例外を発生させる。

プログラムは挿入後の 16 ビット数を返す必要があります。

## 要件

プログラムは以下の要件を満たす必要があります。

- `j` は `i` より大きい必要がある。
- `i` から `j` までに `m` を格納するスペースが足りる必要がある。
- 入力は有効でなければならない。
- プログラムはメモリに収まる必要がある。

## 使い方の例

以下は、プログラムの使い方の例です。

```txt
i      = 2
j      = 6
n      = 0000 0100 0000 0000
m      = 0000 0000 0001 0011
result = 0000 0100 0100 1100
```

この例では、`m` を `n` に挿入して、`m` がビット `j=6` から始まり、ビット `i=2` で終わるようにします。結果の 16 ビット数は `0000 0100 0100 1100` です。
