# Кратчайший путь в невзвешенном направленном графе

## Задача

Дан направленный граф без весов ребер. Найти кратчайший путь между двумя узлами.

## Требования

Для решения этой задачи необходимо выполнить следующие требования:

- Граф является направленным.
- Граф является невзвешенным.
- Классы `Graph` и `Node` уже доступны.
- Входными данными являются два узла.
- Выходом является список ключей узлов, составляющих кратчайший путь.
- Если пути нет, вернуть `None`.
- Граф связный.
- Входные данные действительны.
- Алгоритм должен помещаться в память.

## Пример использования

Предположим, что у нас есть следующий граф:

```
graph.add_edge(0, 1)
graph.add_edge(0, 4)
graph.add_edge(0, 5)
graph.add_edge(1, 3)
graph.add_edge(1, 4)
graph.add_edge(2, 1)
graph.add_edge(3, 2)
graph.add_edge(3, 4)
```

Мы можем найти кратчайший путь между двумя узлами с использованием функции `search_path`:

- `search_path(start=0, end=2) -> [0, 1, 3, 2]`
- `search_path(start=0, end=0) -> [0]`
- `search_path(start=4, end=5) -> None`
