# グラフの幅優先探索

## 問題

グラフ上で幅優先探索（BFS）を実装するには、与えられたソース頂点から始めて、幅優先順序でグラフの全ての頂点を訪問する必要があります。このアルゴリズムは、まずソース頂点を訪問し、次にそのすべての隣接頂点を訪問し、その隣接頂点のすべての隣接頂点を訪問するという具合に動作します。頂点を訪問する順序は重要で、グラフ内のソース頂点から他のすべての頂点までの最短経路を決定します。

## 要件

グラフ上でBFSを実装するには、以下の要件を満たす必要があります。

- グラフは有向グラフです。
- グラフとノードのクラスは既に存在します。
- グラフは連結されています。
- 入力は有効です。
- アルゴリズムはメモリに収まります。

## 使い方の例

次のエッジを持つグラフがあるとします。

```
graph.add_edge(0, 1, 5)
graph.add_edge(0, 4, 3)
graph.add_edge(0, 5, 2)
graph.add_edge(1, 3, 5)
graph.add_edge(1, 4, 4)
graph.add_edge(2, 1, 6)
graph.add_edge(3, 2, 7)
graph.add_edge(3, 4, 8)
```

頂点0からBFSを始めると、訪問されるノードの順序は[0, 1, 4, 5, 3, 2]になります。これは、まず頂点0が訪問され、次にその隣接頂点である1、4、5が訪問され、次に1の隣接頂点（3と4）が訪問され、そして3の隣接頂点（2）が訪問されることを意味します。
