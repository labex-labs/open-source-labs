# 图的广度优先搜索

## 问题

在图上实现广度优先搜索（BFS）需要从给定的源顶点开始，以广度优先的顺序访问图的所有顶点。该算法的工作方式是先访问源顶点，然后访问其所有邻居，接着访问其邻居的所有邻居，依此类推。顶点的访问顺序很重要，因为它决定了从源顶点到图中所有其他顶点的最短路径。

## 要求

要在图上实现 BFS，必须满足以下要求：

- 图是有向的。
- 图类和节点类已经可用。
- 图是连通的。
- 输入是有效的。
- 算法适合内存。

## 示例用法

假设我们有一个具有以下边的图：

```
graph.add_edge(0, 1, 5)
graph.add_edge(0, 4, 3)
graph.add_edge(0, 5, 2)
graph.add_edge(1, 3, 5)
graph.add_edge(1, 4, 4)
graph.add_edge(2, 1, 6)
graph.add_edge(3, 2, 7)
graph.add_edge(3, 4, 8)
```

如果我们从顶点 0 开始进行 BFS，访问节点的顺序将是 [0, 1, 4, 5, 3, 2]。这意味着首先访问顶点 0，接着是它的邻居 1、4 和 5，然后是 1 的邻居（3 和 4），再然后是 3 的邻居（2）。
