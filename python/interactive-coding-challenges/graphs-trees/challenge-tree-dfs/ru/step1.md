# Обход дерева в глубину (DFS)

## Задача

Реализовать обход в глубину (in-order, pre-order, post-order) для двоичного дерева. При in-order обходе мы сначала обходим левое поддерево, затем текущий узел, а затем правое поддерево. При pre-order обходе мы сначала обходим текущий узел, затем левое поддерево, а затем правое поддерево. При post-order обходе мы сначала обходим левое поддерево, затем правое поддерево, а затем текущий узел.

## Требования

Для решения этого испытания нам нужно удовлетворить следующим требованиям:

- Можем предположить, что у нас уже есть класс Node с методом insert.
- Когда обрабатываем каждый узел, мы должны вызвать метод ввода `visit_func` для этого узла.
- Можем предположить, что это помещается в память.

## Пример использования

Вот несколько примеров использования алгоритма DFS:

### In-order обход

Для двоичного дерева с значениями 5, 2, 8, 1 и 3 in-order обход будет 1, 2, 3, 5, 8. Для двоичного дерева с значениями 1, 2, 3, 4 и 5 in-order обход будет 1, 2, 3, 4, 5.

### Pre-order обход

Для двоичного дерева с значениями 5, 2, 8, 1 и 3 pre-order обход будет 5, 2, 1, 3, 8. Для двоичного дерева с значениями 1, 2, 3, 4 и 5 pre-order обход будет 1, 2, 3, 4, 5.

### Post-order обход

Для двоичного дерева с значениями 5, 2, 8, 1 и 3 post-order обход будет 1, 3, 2, 8, 5. Для двоичного дерева с значениями 1, 2, 3, 4 и 5 post-order обход будет 5, 4, 3, 2, 1.
