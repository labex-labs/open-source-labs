# グラフのビルド順序

## 問題

プロジェクトとその依存関係のリストが与えられたとき、有効なビルド順序を見つける必要があります。ビルド順序とは、各プロジェクトがそれに依存するプロジェクトよりも前に表示されるプロジェクトのリストです。

## 要件

この問題を解決するには、次の要件を考慮する必要があります。

- 入力には循環グラフが含まれている場合があります。
- 既にGraphとNodeクラスがあると仮定できます。
- グラフが連結されていると仮定できます。
- 入力が有効であると仮定できます。
- 問題がメモリに収まると仮定できます。

## 例

次のプロジェクトと依存関係があるとします。

- プロジェクト：a、b、c、d、e、f、g
- 依存関係：(d, g)、(f, c)、(f, b)、(f, a)、(c, a)、(b, a)、(a, e)、(b, e)

出力は次のようになります。d、f、c、b、g、a、e

エッジの方向は下向きであることに注意してください。つまり、プロジェクトはその下にあるプロジェクトに依存しています。

```txt
    f     d
   /|\    |
  c | b   g
   \|/|
    a |
    |/
    e
```

入力に循環グラフが含まれている場合、出力はNoneになります。
