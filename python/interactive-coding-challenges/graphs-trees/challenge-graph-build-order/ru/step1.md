# Построение графа в порядке

## Задача

Даны список проектов и их зависимости, и нам нужно найти допустимый порядок сборки. Порядок сборки - это список проектов, в котором каждый проект появляется перед любым проектом, который от него зависит.

## Требования

Для решения этой задачи необходимо учитывать следующие требования:

- Входные данные могут содержать циклический граф.
- Можно предположить, что у нас уже есть классы Graph и Node.
- Можно предположить, что граф связный.
- Можно предположить, что входные данные валидны.
- Можно предположить, что задача помещается в память.

## Пример

Предположим, что у нас есть следующие проекты и зависимости:

- проекты: a, b, c, d, e, f, g
- зависимости: (d, g), (f, c), (f, b), (f, a), (c, a), (b, a), (a, e), (b, e)

Результат должен быть: d, f, c, b, g, a, e

Обратите внимание, что направление ребра идет вниз, что означает, что проект зависит от проектов ниже него.

```txt
    f     d
   /|\    |
  c | b   g
   \|/|
    a |
    |/
    e
```

Если входные данные содержат циклический граф, результат должен быть None.
