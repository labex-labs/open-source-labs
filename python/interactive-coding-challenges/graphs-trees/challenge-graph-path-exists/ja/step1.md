# グラフ内のパスの存在

## 問題

有向グラフと2つのノードが与えられたとき、2つのノード間にパスが存在するかどうかを判断するPython関数を書きます。

## 要件

この問題を解くには、次の仮定を行う必要があります。

- グラフは有向です。
- 既にGraphとNodeクラスがあります。
- グラフは連結されています。
- 入力は有効です。
- ソリューションはメモリに収まります。

## 例の使用法

次のようなグラフがあるとします。

```
graph.add_edge(0, 1, 5)
graph.add_edge(0, 4, 3)
graph.add_edge(0, 5, 2)
graph.add_edge(1, 3, 5)
graph.add_edge(1, 4, 4)
graph.add_edge(2, 1, 6)
graph.add_edge(3, 2, 7)
graph.add_edge(3, 4, 8)
```

2つのノード間にパスが存在するかどうかを判断するには、次の関数を使用できます。

```
search_path(start=0, end=2) -> True
search_path(start=0, end=0) -> True
search_path(start=4, end=5) -> False
```

最初の2つの関数呼び出しはTrueを返します。なぜなら、開始ノードと終了ノードの間にパスが存在するからです。最後の関数呼び出しはFalseを返します。なぜなら、開始ノードと終了ノードの間にパスが存在しないからです。
