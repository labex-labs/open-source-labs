# 二分探索木の2番目に大きいノード

## 問題

二分探索木が与えられたとき、木の2番目に大きいノードを見つけます。入力がNoneまたは単一のノードの場合、例外を発生させる必要があります。

この問題を解くために、特定の順序で木を探索し、これまでに見た2番目に大きいノードを追跡します。まず、根ノードの右部分木を探索します。右部分木がNoneの場合、最大のノードは根ノード自体です。右部分木がNoneでない場合、右部分木を探索し続けて、右の子を持たないノードに到達するまで進めます。この時点で、木の最大のノードはこのノードの親です。この親ノードが左の子を持つ場合、2番目に大きいノードは親ノードの左部分木の最大のノードです。親ノードが左の子を持たない場合、2番目に大きいノードは親ノード自体です。

## 要件

このチャレンジの要件は以下の通りです。

- 入力がNoneまたは単一のノードの場合、例外を発生させる必要があります。
  - None入力の場合、TypeErrorを発生させる必要があります。
  - 単一のノード入力の場合、ValueErrorを発生させる必要があります。
- 既にinsertメソッドを持つNodeクラスがあると仮定できます。
- この問題はメモリに収まると仮定できます。

## 使い方の例

この関数の使い方の例をいくつか示します。

- Noneまたは単一のノード -> 例外

```txt
入力:
                _10_
              _/    \_
             5        15
            / \       / \
           3   8     12  20
          /     \         \
         2       4        30

出力: 20

入力:
                 10
                 /
                5
               / \
              3   7
出力: 7
```
