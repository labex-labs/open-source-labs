# Проблема

В Python почти все, касающееся классов и объектов, является _открытым_.

- Вы можете легко изучать внутренности объекта.
- Вы можете в любой момент изменять вещи.
- Нет сильной концепции контроля доступа (т.е., приватных членов класса)

Это проблема, когда вы пытаетесь изолировать детали _внутренней реализации_.
