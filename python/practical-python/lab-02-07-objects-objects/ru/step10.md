# Все является объектом

Числа, строки, списки, функции, исключения, классы, экземпляры и т.д. все являются объектами. Это означает, что все объекты, которые можно назвать, могут быть переданы в качестве данных, помещены в контейнеры и т.д., без каких-либо ограничений. Нет _особых_ видов объектов. Иногда говорят, что все объекты являются "первоклассными".

Простой пример:

```python
>>> import math
>>> items = [abs, math, ValueError ]
>>> items
[<built-in function abs>,
  <module'math' (builtin)>,
  <type 'exceptions.ValueError'>]
>>> items[0](-45)
45
>>> items[1].sqrt(2)
1.4142135623730951
>>> try:
        x = int('not a number')
    except items[2]:
        print('Failed!')
Failed!
>>>
```

Здесь `items` - это список, содержащий функцию, модуль и исключение. Вы можете напрямую использовать элементы списка вместо исходных имен:

```python
items[0](-45)       # abs
items[1].sqrt(2)    # math
except items[2]:    # ValueError
```

С большой властью приходит ответственность. Просто потому, что вы можете это сделать, не означает, что вы должны.

В этом наборе упражнений мы рассмотрим некоторые аспекты власти, которая присуща первоклассным объектам.
